Contributing to BOSC Community Library

Thank you for helping build a professional Open Source environment!

Technical Instructions
1. Identify an Issue: Browse our open issues and comment to be assigned.
2. Branching: Create a dedicated feature branch for your work.
3. Pull Requests: Use our PR template and link the relevant issue.
4. Peer Review: All changes require a peer review comment before merging.

Standards
- Ensure all code is documented.
- Maintain zero merge conflicts with the main branch.
